# RPG 冒险 · 回合制角色扮演游戏

> 软件工程实训项目 · 基于 Godot 4.3 开发的 2D 像素风回合制 RPG

---

## 一、项目概述

一款经典的 2D 像素风回合制角色扮演游戏。玩家操控一名冒险者，从村庄出发，在地图上自由探索，接触怪物后自动进入回合制战斗界面，通过普攻、技能、道具、逃跑四种指令与怪物对战，获取经验值升级、解锁更强技能，装备武器防具提升战斗力，最终挑战隐藏的强力 BOSS。

**核心玩法循环**：`地图探索 → 遇敌触发回合制战斗 → 战斗结算获取经验 → 升级解锁新技能 → 继续探索更强敌人 → 挑战 BOSS`

---

## 二、游戏特色

| 系统 | 说明 |
|------|------|
| **主菜单** | 新的冒险 / 加载存档 / 退出游戏，带自动缩放的背景图与背景音乐 |
| **单人冒险** | 一名剑士造型的冒险者，操作简单聚焦战斗策略 |
| **回合制战斗** | 玩家回合 → 敌人回合交替，支持技能 / 药水 / 逃跑 |
| **技能系统** | 斩击 / 重斩 / 破甲斩 / 怒斩苍穹，带震动特效与剑气光效 |
| **蓝量消耗** | 强力技能消耗 MP，需合理规划使用或蓝瓶补给 |
| **等级成长** | 击杀怪物获得经验，升级自动强化 HP / MP / 攻击 / 防御 |
| **装备系统** | 武器 + 护甲 + 饰品三件套，提供属性加成 |
| **道具系统** | 血瓶 / 蓝瓶，战斗中可使用回血回蓝 |
| **怪物图鉴** | 史莱姆、蘑菇怪、巨鼠、蝙蝠、骷髅兵、岩石巨人、骨骑士、史莱姆王、亡灵法师、死亡使者 |

| **天使拯救** | 玩家死亡后可选择复活，返回村庄恢复满血满蓝 |
| **存档读档** | 支持多个存档位，可在主菜单加载存档 |

## 八、技术架构

### 8.1 引擎与语言

- **引擎**：Godot 4.3
- **语言**：GDScript
- **渲染**：Forward+
- **美术风格**：2D 像素风（像素精灵 + SpriteFrames 动画）

### 8.2 核心模块设计

**全局单例（Autoload）**：
- `GameData` — 玩家所有持久化属性（等级、经验、HP、MP、装备、道具、已击败怪物位置、战斗前坐标、存档系统）
- `BattleManager` — 战斗流程控制（回合切换、技能释放、经验结算、掉落奖励、场景切换回地图、天使拯救）

**战斗流程**：
```
怪物接触玩家 → emit_signal("enter_battle")
→ BattleManager 记录玩家位置与怪物数据
→ 切换到 combat_manager.tscn
→ 复制怪物精灵帧并重新计算脚对齐
→ 战斗场景按视口比例缩放背景与角色
→ 玩家回合 / 敌人回合循环（含技能震动与剑气光效）
→ 战斗结束：发放经验 + 金币 + 掉落 → 升级判定 → 刷新技能解锁
→ 切回原地图场景，玩家在战斗前位置恢复
```

**动画与对齐系统**：
- 玩家 / 怪物动画使用 SpriteFrames + AnimatedSprite2D
- 脚对齐（Foot Alignment）：根据动画帧大小自动计算 offset，让角色脚底对齐节点原点，解决不同怪物帧大小不一致导致的显示不对齐问题
- 精灵帧动态复制：怪物进入战斗时从原场景复制精灵帧，保证动画一致性
- 朝向翻转：战斗中怪物统一朝左（面向玩家），特殊怪物（如骷髅）保持默认朝向

**战斗视觉特效**（combat_manager.gd）：
- 重斩：角色轻微震动
- 破甲斩：弧形重劈气刃 + 盔甲碎裂白光 + 金属碎片飞散 + 怪物厚重震动
- 怒斩苍穹：全屏强光爆发 + 天空劈下弧形气刃 + 3 条弧形斜斩 + 火焰流光 + 十字交叉弧形斩 + 冲击波扩散 + 怪物剧烈震动

## 九、运行方式

### 9.1 从 Godot 编辑器运行

1. 安装 [Godot 4.3](https://godotengine.org/download) 或更高版本
2. 打开 Godot，选择 `Import` → 选中 `project.godot`
3. 点击右上角 ▶ 运行按钮，从 `start.tscn` 启动
4. 主菜单按 **"新的冒险"** 进入序章，或 **"加载存档"** 读取存档，或 **"退出游戏"** 关闭程序

### 9.2 游戏流程

启动画面 → 开场剧情 → 村庄自由移动 → 进入森林/亡灵区探索 → 碰到怪物进入战斗 → 战斗胜利获取经验与金币 → 升级解锁新技能 → 回村庄补给 → 继续挑战更强敌人 → 最终挑战 BOSS

## 十一、开发要点记录

- **单例设计**：`GameData` 与 `BattleManager` 均为 Autoload，跨场景常驻
- **战斗位置还原**：BattleManager 在进入战斗前记录 `player_return_position`，切回地图时恢复
- **已击败怪物不重生**：`GameData.defeated_monster_positions` 记录击败位置，地图重载时自动过滤
- **技能等级锁定**：`GameData` 中定义各技能的解锁等级，UI 实时刷新按钮状态
- **动画适配**：怪物进入战斗后，源怪物的精灵帧、position、scale 会同步复制到战斗场景中的 monster-battler，再由 `reapply_foot_alignment` 重新计算脚对齐
- **MP 管理**：蓝条与血条独立，战斗中技能消耗 MP，不足时仍可点击但会提示 MP 不足
- **场景缩放**：战斗场景按 `max(viewport.width/设计宽, viewport.height/设计高)` 统一缩放背景与角色，保证全屏无空白
- **天使拯救**：玩家死亡后显示拯救面板，回满 HP/MP 并传送回村庄指定位置，确保游戏可继续进行
- **主菜单退出**：主菜单提供"退出游戏"按钮，点击后停止背景音乐并退出程序
