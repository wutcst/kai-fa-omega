软件工程实训任务二：小组协同开发（回合制RPG游戏）
一、整体项目总架构
场景体系：村庄安全场景 + 多层地下城冒险场景
角色体系：初始剑士（开局）→ 解锁游侠、盾骑士双职业
核心系统：角色升级、属性成长、职业技能、回合制战斗、怪物AI、战斗结算、村庄NPC交互、存档读档
玩法流程：村庄补给/接任务→进入地下城遇敌触发回合制战斗→打怪升级获取道具奖励→回村回血休整、修缮补给
项目类型变更：原即时动作打怪游戏，调整为经典回合制RPG游戏，核心战斗逻辑改为回合轮流对战模式，其余整体架构、场景、UI体系保持不变。
二、分工安排（各成员负责板块不变，内容适配回合制玩法）
组长赵灿：主程序（核心逻辑+回合制战斗系统）
负责：所有游戏底层逻辑、回合制战斗、职业体系、角色升级、数值配置、全局数据与存档
工作内容
1. 角色基础框架
玩家实体制作、血量、蓝量、攻击力、防御、速度等基础属性系统搭建；完成三职业切换系统开发，包含初始剑士、高速远程游侠、高防近战盾骑士；搭建完整角色等级经验体系，实现升级自动属性加成、升级弹窗提示功能，适配回合制战斗数值规则。
2. 回合制战斗系统（项目核心）
实现地下城随机遇敌、定点遇敌触发回合制战斗场景切换；搭建完整回合制战斗流程，实现玩家回合、怪物回合循环交替机制；编写通用普攻逻辑与三职业专属技能逻辑，配置技能蓝量消耗、冷却机制、特效对应逻辑；设计回合制怪物AI，实现怪物智能普攻、释放技能、防御等行为判定；完善暴击、闪避、伤害计算公式与数值判定；搭建战斗胜负判定机制，完成战后经验、金币、道具自动结算功能。
3. 数据与存档
负责全局游戏数据统筹管理，搭建稳定的本地存档读档系统，支持角色等级、职业属性、血量蓝量、背包道具、战斗进度等核心数据的保存与读取，保障游戏数据连贯性。
4. 通用工具脚本
开发适配回合制战斗的血条、蓝条、经验条UI联动逻辑；实现战斗伤害数字飘字、暴击提示、闪避提示、回合状态提示等通用效果脚本，适配回合制战斗展示需求。
组员许世豪：场景搭建+地图+碰撞+关卡设计
负责：村庄全场景、地下城多层地图、地图交互、角色寻路、场景跳转、关卡氛围搭建
工作内容
1. 像素瓦片地图制作
完成完整村庄场景铺设，包含房屋、道路、水井、草地、出入口等基础场景元素；分层设计地下城关卡地图，分为第一层小怪区域、中层精英怪区域、深层BOSS专属房间；完成所有地图的瓦片碰撞层、角色行走区域、障碍物阻挡区域精细化设置，保障回合制探索场景的行走合理性。
2. 场景功能实现
搭建村庄与地下城双向场景传送跳转功能；设置地图专属触发区域，实现进入副本、离开副本、遇敌触发战斗等场景交互逻辑；实现角色地面自由行走、地图视角相机跟随功能，适配回合制探索玩法。
3. 环境与氛围
负责全场景装饰物、植被、建筑的摆放与优化；打造地下城专属黑暗氛围感，调试基础光源效果，区分村庄安全场景与地下城冒险场景的氛围差异。
4. 简单NPC点位布置
在村庄场景预留并固定商人、旅馆、任务NPC的点位，对接后续NPC交互功能，保障场景功能完整性。
组员张凌丰：UI界面+美术资源+交互+剧情文案
负责：所有游戏界面、图标资源、动画特效、剧情文案、音效适配、NPC交互逻辑
工作内容
1. 全套游戏UI制作
设计制作游戏主界面、角色状态栏、背包界面、职业选择界面、升级提示界面；重点适配制作回合制战斗专属UI，包含战斗指令按钮（普攻、技能、道具、防御）、双方战斗状态栏、回合进度提示栏；制作商店购买界面、旅馆休息回血界面等功能UI。
2. 动画与美术适配
完成三职业角色行走、待机、回合攻击像素动画的导入与适配调试；优化怪物攻击、受击、死亡回合制动画效果；制作适配回合制节奏的普攻特效、职业技能特效，匹配战斗回合节奏。
3. 剧情与交互
编写村庄NPC对话系统，完善对话交互逻辑；对接程序完成旅馆休息回血、商店道具买卖的交互功能；撰写游戏世界观文案、任务文案、战斗提示文案、新手引导文案，适配回合制游戏玩法说明。
4. 音效与体验优化
挂载调试场景行走、角色升级、普攻、技能释放、怪物击败等全套音效；新增回合切换、战斗触发、胜负结算专属音效；优化游戏弹窗提示、文字提示样式，提升回合制游戏整体游玩体验。
三、团队统一变量命名规范
血量：hp / 最大血量：max_hp
蓝量：mp / 最大蓝量：max_mp
攻击：attack / 防御：defense
速度：speed / 暴击：crit
职业：warrior（剑士）、ranger（游侠）、knight（盾骑士）
