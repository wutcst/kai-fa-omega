# RPG 冒险 · 回合制角色扮演游戏

> 软件工程实训项目 · 基于 Godot 4.3 开发的 2D 像素风回合制 RPG

---

## 一、项目概述

一款经典的 2D 像素风回合制角色扮演游戏。玩家操控一名冒险者，从村庄出发，在地图上自由探索，接触怪物后自动进入回合制战斗界面，通过普攻、技能、道具、逃跑四种指令与怪物对战，获取经验值升级、解锁更强技能，装备武器防具提升战斗力，最终挑战亡灵法师 BOSS。

**核心玩法循环**：`地图探索 → 遇敌触发回合制战斗 → 战斗结算获取经验与掉落 → 升级解锁新技能 → 继续探索更强敌人 → 挑战最终 BOSS`

---

## 二、游戏特色

| 系统 | 说明 |
|------|------|
| **主菜单** | 新的冒险 / 加载存档 / 退出游戏，带自动缩放的背景图与背景音乐 |
| **开场剧情** | 黑底白字逐句播放，按 E 继续，奠定世界观 |
| **结局剧情** | 击败亡灵法师后触发，黑底暖色字幕逐句播放，按 E 回到主菜单 |
| **单人冒险** | 一名剑士造型的冒险者，操作简单聚焦战斗策略 |
| **回合制战斗** | 玩家回合 → 敌人回合交替，支持技能 / 药水 / 逃跑 |
| **技能系统** | 斩击 / 重斩 / 破甲斩 / 怒斩苍穹，带震动特效与剑气光效 |
| **蓝量消耗** | 强力技能消耗 MP，需合理规划使用或蓝瓶补给 |
| **等级上限** | 最高 Lv.16，满级后不再获得经验 |
| **装备系统** | 武器 + 护甲 + 饰品三件套，5 个套装等级，提供属性加成 |
| **掉落系统** | 普通怪物概率掉落装备/药水，BOSS 保底掉落 1 件装备 + 50% 再掉 1 件 |
| **道具系统** | 血瓶（回 50 HP）/ 蓝瓶（回 30 MP），战斗中可使用 |
| **食物系统** | 商人出售苹果、西瓜、草莓、樱桃等，提供回血/回蓝/永久属性加成 |
| **怪物图鉴** | 蘑菇怪、史莱姆、巨鼠、蝙蝠、骷髅兵、岩石巨人、骨骑士、史莱姆王、死亡使者、亡灵法师 |
| **地图系统** | 村庄（安全区）、森林、亡灵区、最终战斗区，通过传送门切换 |
| **NPC 对话** | 村长、商人、旅馆老板等多名 NPC，提供商店、休息、剧情对话 |
| **BOSS 战** | 多名 BOSS 级敌人，拥有独特属性与战斗体验 |
| **亡灵法师** | 最终 BOSS，3 回合周期循环（召唤随从 → 普通攻击 → 技能攻击），逃跑禁止 |
| **召唤随从** | 亡灵法师可召唤随从，随从替 BOSS 抵挡一次任意攻击后死亡 |
| **天使拯救** | 玩家死亡后可选择复活，返回村庄恢复满血满蓝 |
| **存档读档** | 支持多个存档位，可在主菜单加载存档，随时随地保存 |

## 六、操作说明

| 按键 | 地图操作 |
|------|----------|
| W / ↑ | 向上移动 |
| S / ↓ | 向下移动 |
| A / ← | 向左移动 |
| D / → | 向右移动 |
| Q | 打开物品栏 / 人物信息面板 |
| E | 序章/结局中继续对话 |
| 靠近 NPC | 触发对话 / 商店 / 旅馆 |

|------|----------|

## 八、技术架构

### 8.1 引擎与语言

- **引擎**：Godot 4.3
- **语言**：GDScript
- **渲染**：Forward+
- **美术风格**：2D 像素风（像素精灵 + SpriteFrames 动画）

### 8.2 核心模块设计

**全局单例（Autoload）**：
- `GameData` — 玩家所有持久化属性（等级、经验、HP、MP、装备、道具、已击败怪物位置、战斗前坐标、存档系统）
- `BattleManager` — 战斗流程控制（回合切换、技能释放、经验结算、掉落奖励、场景切换回地图、天使拯救、Boss 逃跑拦截）

**战斗流程**：
```
怪物接触玩家 → emit_signal("enter_battle")
→ BattleManager 记录玩家位置与怪物数据
→ 切换到 combat_manager.tscn
→ 复制怪物精灵帧并重新计算脚对齐
→ 战斗场景按视口比例缩放背景与角色
→ 玩家回合 / 敌人回合循环（含技能震动与剑气光效）
→ 亡灵法师特殊回合：3 回合循环（召唤随从 → 普通攻击 → 技能攻击）
→ 战斗结束：发放经验 + 金币 + 掉落 → 升级判定 → 刷新技能解锁
→ 击败亡灵法师：跳转结局场景
→ 其他情况：切回原地图场景，玩家在战斗前位置恢复
```

**动画与对齐系统**：
- 玩家 / 怪物动画使用 SpriteFrames + AnimatedSprite2D
- 脚对齐（Foot Alignment）：根据动画帧大小自动计算 offset，让角色脚底对齐节点原点
- 精灵帧动态复制：怪物进入战斗时从原场景复制精灵帧，保证动画一致性
- 朝向翻转：战斗中怪物统一朝左（面向玩家），特殊怪物保持默认朝向
- MONSTER_Y_OFFSETS 表：每个怪物有独立的 Y 轴微调偏移，全屏缩放时自动适配

**战斗视觉特效**（combat_manager.gd）：
- 重斩：角色轻微震动
- 破甲斩：弧形重劈气刃 + 盔甲碎裂白光 + 金属碎片飞散 + 怪物厚重震动
- 怒斩苍穹：全屏强光爆发 + 天空劈下弧形气刃 + 3 条弧形斜斩 + 火焰流光 + 十字交叉弧形斩 + 冲击波扩散 + 怪物剧烈震动

**掉落系统**：
- 每个怪物有独立的金币区间和装备层级配置
- 普通怪物 45% 概率掉落装备，25% 概率掉落药水
- BOSS 保底掉落 1 件装备，额外 50% 概率再掉 1 件
- 装备按怪物 tier 范围随机选取

**亡灵法师 AI 系统**：
- 3 回合周期：召唤随从 → 普通攻击（带黑暗弧形波特效） → 技能攻击（固定 40 伤害）
- 随从机制：替 BOSS 抵挡一次任意攻击，无血量，被击中后直接死亡
- 逃跑拦截：Boss 战中点击逃跑弹出提示面板，按 E 关闭后视为逃跑失败

## 九、运行方式

### 9.1 从 Godot 编辑器运行

1. 安装 [Godot 4.3](https://godotengine.org/download) 或更高版本
2. 打开 Godot，选择 `Import` → 选中 `project.godot`
3. 点击右上角 ▶ 运行按钮，从 `start.tscn` 启动
4. 主菜单按 **"新的冒险"** 进入序章，或 **"加载存档"** 读取存档，或 **"退出游戏"** 关闭程序

### 9.2 游戏流程

```
启动画面 → 开场剧情 → 村庄自由移动 → 进入森林/亡灵区探索
→ 碰到怪物进入战斗 → 战斗胜利获取经验与掉落 → 升级解锁新技能
→ 回村庄补给 → 继续挑战更强敌人 → 进入最终战斗区
→ 击败亡灵法师 → 结局剧情 → 回到主菜单
```

## 十一、开发要点记录

- **单例设计**：`GameData` 与 `BattleManager` 均为 Autoload，跨场景常驻
- **战斗位置还原**：BattleManager 在进入战斗前记录 `player_return_position`，切回地图时恢复
- **已击败怪物不重生**：`GameData.defeated_monster_positions` 记录击败位置，地图重载时自动过滤
- **BOSS 击败不重生**：`GameData.defeated_boss_positions` 记录 BOSS 击败位置，进入最终战斗区时自动清除确保 BOSS 登场
- **技能等级锁定**：`GameData.SKILL_REQ` 定义各技能解锁等级，UI 实时刷新按钮状态
- **动画适配**：怪物进入战斗后，源怪物的精灵帧、position、scale 会同步复制到战斗场景，再由 `reapply_foot_alignment` 重新计算脚对齐
- **MP 管理**：蓝条与血条独立，战斗中技能消耗 MP，不足时仍可点击但会提示 MP 不足
- **场景缩放**：战斗场景按 `max(viewport.width/设计宽, viewport.height/设计高)` 统一缩放背景与角色，保证全屏无空白
- **天使拯救**：玩家死亡后显示拯救面板，回满 HP/MP 并传送回村庄指定位置，确保游戏可继续进行
- **主菜单退出**：主菜单提供"退出游戏"按钮，点击后停止背景音乐并退出程序
- **Boss 逃跑拦截**：Boss 战中点击逃跑弹出"勇者，此地凶险，无法逃离"提示面板，按 E 关闭后自动视为逃跑失败
- **召唤物定位**：随从添加到 EnemyTeam 同级节点，精灵缩放和 Y 偏移从亡灵法师动态复制，避免窗口缩放导致的错位
- **结局场景**：击败亡灵法师后自动跳转结局，黑底暖色字幕逐句播放，按 E 回到主菜单
