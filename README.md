哦# RPG-603game · 回合制角色扮演游戏

> 软件工程实训项目 · 基于 Godot 4.3 开发的 2D 像素风回合制 RPG

---

## 一、项目概述

一款经典的 2D 像素风回合制角色扮演游戏。玩家操控一名冒险者，在地图上自由探索，接触怪物后自动进入回合制战斗界面，通过普攻、技能、道具、逃跑四种指令与怪物对战，获取经验值升级、解锁更强技能，装备武器防具提升战斗力。

**核心玩法：** 地图探索 → 遇敌触发回合制战斗 → 战斗结算获取经验 → 升级解锁新技能 → 继续探索更强敌人

---

## 二、游戏特色

| 系统 | 说明 |
|------|------|
| 单人冒险 | 一名冒险者（剑士造型），操作简单聚焦战斗策略 |

| 道具栏 | 血瓶/蓝瓶，战斗中可使用回血回蓝 |
| 六种怪物 | 史莱姆、巨鼠、蝙蝠、哥布林、骷髅兵、恶龙，难度梯度 |

## 七、技术架构

### 7.1 引擎与语言

- **引擎：** Godot 4.3
- **语言：** GDScript
- **渲染：** Forward+
- **美术风格：** 2D 像素风（像素精灵 + SpriteFrames 动画）

### 7.2 核心模块设计

**全局单例（Autoload）：**
- `GameData` — 玩家所有持久化属性（等级、经验、HP、MP、装备、道具、已击败怪物位置、战斗前坐标）
- `BattleManager` — 战斗流程控制（回合切换、技能释放、经验结算、场景切换回地图）

**战斗流程：**
```
怪物接触玩家 → emit_signal("enter_battle")
→ BattleManager 记录玩家位置与怪物数据
→ 切换到 combat_manager.tscn
→ 玩家回合 / 敌人回合循环
→ 战斗结束：发放经验 → 升级判定 → 刷新技能解锁
→ 切回 game_scene.tscn，玩家在战斗前位置恢复
```

**动画系统：**
- 玩家：swordsman_idle / walk / hit / attack / heavyhit / hurt / death（动态生成 heavyhit）
- 怪物：monster_name_idle / walk / attack / hurt / death
- 动画时长由 SpriteFrames 帧率自动计算，避免"动画播完但等待时间未到"的卡顿

## 八、运行方式

### 8.1 从 Godot 编辑器运行

1. 安装 [Godot 4.3](https://godotengine.org/download) 或更高版本
2. 打开 Godot，选择 `Import` → 选中 `project.godot`
3. 点击右上角 ▶ 运行按钮，从 `start.tscn` 启动

### 8.2 游戏流程

启动画面 → 开始游戏 → 地图自由移动 → 碰到怪物进入战斗 → 战斗胜利获取经验 → 升级解锁新技能 → 继续挑战更强敌人

## 十、开发要点记录

- **单例设计：** `GameData` 与 `BattleManager` 均为 Autoload，跨场景常驻
- **战斗位置还原：** BattleManager 在进入战斗前记录 `player_return_position`，切回地图时恢复
- **已击败怪物不重生：** `GameData.defeated_monster_positions` 记录击败位置，地图重载时自动过滤
- **技能等级锁定：** `SKILL_REQ` 常量数组定义各技能的解锁等级，UI 实时刷新按钮状态
- **动画适配：** 玩家攻击动画通过 `_create_attack_variants()` 动态从 hit 拆分出普通/蓄力斩
- **MP 管理：** 蓝条与血条独立，战斗中技能消耗 MP，不足时按钮仍可点但会提示 MP 不足
